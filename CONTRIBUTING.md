# Contributing

[한국어](CONTRIBUTING-KR.md)

## Welcome Contributions

- Bug fixes
- Translation improvements
- Documentation corrections
- Small enhancements

## Before Submitting PR

1. Run `npm run build` - must succeed
2. Run `npm run i18n:validate` - must pass
3. Keep changes minimal and focused
4. Test manually in browser

## Not Accepted

- Large feature additions (open Issue first to discuss)
- New dependencies
- Major structural changes
- Changes unrelated to the PR description

## How to Contribute

1. Fork the repository
2. Create a branch (`git checkout -b fix/bug-description`)
3. Make changes
4. Commit with clear message
5. Push and create PR

## Response Time

This is a personal project. Response may be slow. Thank you for your patience.

## Questions

Open an Issue for questions or suggestions.
