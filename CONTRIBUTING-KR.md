# 기여 가이드

[English](CONTRIBUTING.md)

## 환영하는 기여

- 버그 수정
- 번역 개선
- 문서 오류 수정
- 작은 개선사항

## PR 제출 전 확인

1. `npm run build` 실행 - 성공해야 함
2. `npm run i18n:validate` 실행 - 통과해야 함
3. 변경 범위를 최소화
4. 브라우저에서 수동 테스트

## 받지 않는 기여

- 대규모 기능 추가 (먼저 Issue로 논의)
- 새로운 의존성 추가
- 구조 변경
- PR 설명과 무관한 변경

## 기여 방법

1. 저장소 Fork
2. 브랜치 생성 (`git checkout -b fix/버그-설명`)
3. 변경 작업
4. 명확한 메시지로 커밋
5. Push 후 PR 생성

## 응답 시간

개인 프로젝트라 응답이 느릴 수 있습니다. 양해 부탁드립니다.

## 질문

질문이나 제안은 Issue를 열어주세요.
